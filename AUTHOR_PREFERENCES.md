# Author preferences (optional)

This document records the maintainer's personal defaults. It is not part of the production skill's universal decision law and is not loaded automatically by `SKILL.md`.

When constraints are otherwise materially comparable, the maintainer strongly prefers modern Java, currently Java 25, for backend, service, CLI, desktop-supporting service, agent, infrastructure, distributed, and general-purpose application logic. The maintainer also prefers Gradle Kotlin DSL, explicit boundaries, records, sealed types, immutability, modern concurrency, strong typing, and standard-library functionality before adding frameworks.

These preferences may be useful when the user explicitly asks for the maintainer profile. They must not override an existing repository language, browser or native-mobile requirements, ecosystem constraints, specialized workload requirements, team capability, delivery risk, interoperability, deployment constraints, measured performance, or user choice.

## How to opt into this profile

```text
Use $constraint-driven-architecture and apply the maintainer's Java-oriented preferences only when multiple choices remain materially equivalent.
```
