# Constraint-Driven Architecture v0.1.0-alpha.1

First public alpha of Constraint-Driven Architecture, an instruction-only
Codex skill for greenfield architecture, major system evolution, and
repository-level architecture review.

## Included

- `GREENFIELD`, `EVOLUTION`, and `REVIEW` modes
- explicit evidence classification
- smallest-credible-baseline selection
- adversarial removal of unjustified complexity
- proportional architecture output
- security, reliability, performance, contract, ownership, and delivery gates
- offline dual-model evaluation harness
- immutable generation inputs
- blinded evaluator packages

## Current limitations

- no complete scored public evaluation package is currently published
- architecture evaluation remains partly subjective
- the harness does not invoke models
- two-model comparison is not a universal benchmark
- generated outputs are not expected to reproduce byte-for-byte
- the skill may still overengineer or underengineer
- security-critical and formally verified systems require specialist review

## Installation

Install the skill by cloning the repository into:

`~/.agents/skills/constraint-driven-architecture`

See the repository README for Windows, macOS, and Linux commands.

## Status

This is an alpha release.

Installation behavior, evaluation methodology, output structure, and skill
rules may change based on evidence and user feedback.
